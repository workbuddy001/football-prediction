## 第三步半：⭐ 造热检查（V2.8核心新增·Step 2.5）

> **⚠️ 在执行第四步排除法之前，必须先完成造热检查！**
> 造热检查的结果直接影响后续排除判断，不能延后到庄家意图验证阶段。
> 案例：巴列卡诺1-0斯特拉斯堡 — 系统推荐3球黄金信号，但5信号全指向=造热，提前排除3球✅

### 3.5.1 为什么造热检查必须前置？

```
旧版流程（V2.7及以前）：造热检查在第6.5步
  问题：排除法已经排除了大部分选项，造热检查来"事后验证"
  漏洞：当系统推荐某方向且系统排除了其他方向时，
        排除法会自动保留系统推荐方向，造热检查形同虚设

新版流程（V2.8）：造热检查在排除法之前
  优势：先识别哪个方向是陷阱，再执行排除法
  结果：系统推荐方向如果是造热，会和推离方向一样被排除
```

### 3.5.2 造热检查三步法

#### 检查A：系统推荐方向的5信号全指向检查

```
当系统推荐某个方向时（如"让负+3球黄金"），执行5信号检查：

A1: 赔率在黄金/舒适区间？
  3球3.35 ∈ [3.2, 3.5] → 处于玩家"愿意下注"的区间

A2: 近况支持该结果？
  近况6.6极高 → 数据面完美支撑大球

A3: 双命中率不排除？
  21.7% + 28.6% → 双中，统计学支持

A4: 历史比分/系统信号支持？
  2:1/1:2/3:0 → 系统明确提示高命中比分

A5: 变化方向也支持？
  3球↓8.2% → 赔率下降，制造"稳了"的错觉

判断：
  5信号全指向 = 🚨 造热陷阱！→ 排除该方向
  4信号指向 = 警惕，降低置信度
  ≤3信号指向 = 可能不是陷阱
```

#### 检查B：最低赔方向造热检查（注意事项20）

```
如果系统推荐方向恰好也是最低赔方向：
  → 让负1.65超低赔 + 系统推荐"让负+3球"
  → 双重诱导！
  → 排除该方向（最低赔方向被全市场造热=诱导陷阱）
```

#### 检查C：0球反向信号检查

```
如果 0球 ≤ 12 + 系统推荐大球方向：
  → 0球赔率偏低 = 庄家造热0球
  → 实际大球概率比直觉低
  → 0球≤12反向信号触发
  → 削弱大球方向的置信度，不直接排除
```

### 3.5.3 造热检查的处理逻辑

```python
# 按优先级执行，高的直接排除

Step A: 5信号全指向检查
  if 5信号全指向系统推荐方向:
    → 🚨 排除系统推荐方向（被造热）
    → 跳过 Step B/C，直接进入第四步排除法

Step B: 最低赔造热检查
  if 系统推荐方向的最低赔 < 1.70 AND 变化 ≥ 0次（未推离）:
    → 🚨 排除该方向

Step C: 0球反向信号
  if 0球 ≤ 12 AND 系统推荐大球:
    → 削弱置信度，不直接排除
    → 在排除法中优先排除0球（双保险）
```

### 3.5.4 造热检查 vs 6.5步庄家意图验证的区别

| | 造热检查（Step 2.5） | 庄家意图验证（Step 6.5） |
|--|---------------------|------------------------|
| **时机** | 排除法之前 | 排除法之后 |
| **目标** | 审查系统推荐方向是否造热 | 对剩余选项做情境化判断 |
| **范围** | 仅检查系统推荐/最低赔方向 | 所有剩余选项 |
| **结果** | 直接排除陷阱方向 | 微调优先级，不新增排除 |
| **依据** | 5信号全指向 + 最低赔造热 | 宏观方向 + 半全场 + 实盘验证 |

| 变化幅度 | 等级 | 操作 |
|---------|------|------|
| <5% | **弱推离** | ⚠️ 不自动排除，保留观察 |
| 5-10% | 中等推离 | 可排除，但需结合其他信号 |
| ≥10% | **强推离** | 可以排除 |

**⚠️ 变化次数也很关键（1次变化统计意义极弱）**：

| 变化次数 | 可靠性 | 操作 |
|---------|--------|------|
| 1次 | **极弱** | 不论幅度大小，1次变化的推离不可靠，不能草率排除 |
| 2次 | 中等 | 需结合幅度判断 |
| ≥3次 | **较强** | 趋势更可靠 |

**综合判断规则**：
```python
# 只有同时满足"幅度≥5% + 次数≥2次"才算是可靠推离
if 变化幅度 < 5% OR 变化次数 == 1:
    → 不可靠推离，保留该进球数
if 变化幅度 >= 5% AND 变化次数 >= 2:
    → 可靠推离，可考虑排除
if 变化幅度 >= 10% AND 变化次数 >= 2:
    → 强推离，可以排除
```

**案例**：卡利亚里3-2复盘
```
5球：1次↑4.8% → 幅度<5% + 次数=1 → 双重不可靠 → 保留！实际5球打出 ❌教训
6球：2次↑21.1% → 幅度>10% + 次数≥2 → 强推离 → 可以排除 ✅
7球：1次↑16.7% → 幅度大但次数=1 → 不可靠！保留观察
```

**克制推离的含义**：5球1次↑4.8% = 庄家"不得不动但不想动太多" = 5球是庄家在意的方向 = **保留5球**

### 4.1 排除高进球数（4-7球）

**触发条件**：
- 近况正常（2.5-3.5）或偏低
- 高进球数赔率只降1-2次
- 终盘水位不低（不是真实防范）

**结论**：→ **排除4-7球**（庄家假动作/诱大球）

---

### 4.2 排除低进球数（0-1球）

**触发条件**：
- 0球赔率极高（> 20）
- 或1球赔率偏高（> 6.0）
- 近况正常或以上

**结论**：→ **排除0-1球**（进球不会太少）

---

### 4.3 用让球盘辅助排除

| 让球方向 | 庄家看好 | 对应比分 | 排除 |
|---------|---------|---------|------|
| 让胜低赔 | 主队大胜 | 2-0, 3-0, 2-1 | 排除"客队不败" |
| 让平低赔 | 小胜或平 | 1-0, 2-1, 1-1 | 排除"大胜" |
| 让负低赔 | 客队不败 | 0-1, 1-2, 1-1 | 排除"主队大胜" |

**让平赔率特别提示**：
- 让平 < 3.3 → 让平可能打出（需结合前置条件）
- 让平 3.65-3.95 + 客近况好 → 让胜（88.2%置信度）

### 4.4 命中率统计修正（2026-04-28新增）

**⚠️ 必须检查系统提供的命中率统计，作为排除法的修正步骤！**

（曼联VS布伦特福德复盘发现：0球=19统计→3球率66.7%+大3球率0%，直接排除4球，修正了原来的4球备选）

#### 数据来源

系统会根据3球赔率+0球赔率等条件，提供类似以下命中率统计：

```
📊 关注3球+0球=19 命中率统计
预判      样本   小3球(0-2)    恰好3球      大3球(4+)
大3球     3场    1场(33.3%)    2场(66.7%)   0场(0.0%)
```

或：

```
📊 0球<=12反向 命中率统计
预判      样本   小3球(0-2)    恰好3球      大3球(4+)
小3球     13场   6场(46.2%)    3场(23.1%)   4场(30.8%)
```

#### 使用规则

```python
Step 1: 识别命中率统计的预判方向
  → "大3球"预判 = 历史数据中大3球打出更多
  → "小3球"预判 = 历史数据中小3球打出更多

Step 2: 检查各进球数区间的命中率
  → 如果某区间命中率=0% → 强排除该区间！
  → 如果某区间命中率>60% → 强保留该区间！
  → 如果某区间命中率40-60% → 中等保留

Step 3: 用命中率修正排除法结果
  → 如果排除法保留了某进球数，但命中率统计显示0% → 修正为排除
  → 如果排除法弱排除了某进球数，但命中率统计显示>60% → 修正为保留

Step 4: 命中率统计的优先级
  → 命中率统计 > 赔率变化的直觉判断
  → 命中率统计 ≈ 赔率变化的结构分析（同级）
  → 但样本<3场时需降权（标注"样本小"）
```

#### 案例：曼联 VS 布伦特福德（#2039372）

```
0球=19极端区间 → 命中率统计（3场样本）：
  小3球(0-2): 33.3%
  恰好3球:    66.7%  ← 最高！
  大3球(4+):  0.0%   ← 0%！

排除法原结果：3球保留 + 4球保留（4球↓11.1%强防范）

命中率统计修正：
  → 大3球率0% → 排除4球！
  → 3球率66.7% → 3球强保留！

修正后：3球唯一高概率方向 → 推荐2-1（3球主胜）✅ 命中

如果不用命中率统计：
  → 4球↓11.1%强防范 → 会保留4球作为备选 → 降低3球的置信度
  → 命中率统计直接排除4球 → 3球置信度从⭐⭐⭐⭐升级为⭐⭐⭐⭐⭐
```

### 4.6 赔率变化命中率修正（2026-04-28新增）

**⚠️ 用赔率变化命中率验证或修正推离排除的结果！**

（切尔西VS利兹联复盘发现：4球↓8.7%看似"防范"，但赔率变化命中率0%(11场)——11场比赛中从未打过4球！这是"假防范真诱导"。0球↑25%看似"强推离"，但命中率50%(2场)——样本小但高，不应草率排除。）

#### ⚠️ 这段数据是什么？在推理流程中处于什么位置？

**系统界面中"总进球变化"这段数据，是整个推理流程的核心数据源之一。**

```
推理流程中的位置：

第一步  近况区间          → 不使用此数据
第二步  理论vs实际盘口    → 不使用此数据（用0球赔率终盘/初赔）
第三步  赔率变化分析      → ★ 使用"变化次数+方向+幅度"部分 ← 本段数据的前半部分
第四步  5.8-5.19节检查   → 部分使用（变化次数、幅度用于判断招摇/陷阱）
  4.0   变化幅度分级      → ★ 使用"幅度%+次数"判断推离可靠性
  4.1-4.3 排除法         → ★ 使用"方向↑↓+幅度+次数"判断排除
  4.4   规律命中率统计    → 不使用此数据（使用独立的"0球=X命中率统计"）
  4.6   赔率变化命中率    → ★ 使用"命中率%(N场)"部分 ← 本段数据的后半部分
  4.5   排除法汇总        → 综合4.0-4.6的结论
第五步  聚焦推荐          → 不直接使用此数据（用排除法结果）
```

**这段数据由两组独立信息组成，服务于推理流程的不同步骤**：

| 数据 | 示例 | 服务于哪个步骤 | 作用 |
|------|------|---------------|------|
| **变化次数+方向+幅度** | `2次 ↑3.6%` | 第三步→4.0节→4.1-4.3节 | 判断庄家对每个进球数的态度（推离/防范/不动） |
| **历史命中率+样本量** | `0%(4场)` | 4.6节 | 验证上述判断是否真实（假防范/假推离/真信号） |

**⚠️ 关键：两组数据必须配合使用！**

```
只用"变化次数+方向+幅度"（不用命中率）的风险：
  → 4球↓8.7% → 判断"庄家防范4球" → 保留4球
  → 但命中率0%(11场) → 11场从未打出4球 → "防范"是假的！→ 排除4球
  → 没有命中率验证 = 可能被庄家诱导

只用"命中率"（不用变化方向+幅度）的风险：
  → 3球命中率30.7%(88场) → 看起来3球很能打
  → 但3球0次变化+→0% → 庄家完全没动3球 → "不设防"信号
  → 没有变化方向 = 不知道庄家当前态度

正确做法：先看变化判断庄家态度，再用命中率验证这个态度是否真实
```

#### 数据详解：每一行是什么意思？

系统在"赔率变化统计"区域显示以下数据。**以真实比赛为例逐行拆解**：

```
总进球变化
0球  2次 ↑3.6%   0%(4场)     ← 第1行
1球  2次 ↑3.8%   5.7%(35场)  ← 第2行
2球  2次 →0%     15.2%(66场) ← 第3行
3球  0次 →0%     30.7%(88场) ← 第4行
4球  2次 →0%     10.1%(79场) ← 第5行
5球  2次 ↓2.9%   9.5%(21场)  ← 第6行
6球  2次 ↓6.2%   14.3%(7场)  ← 第7行
7球  2次 ↓4.2%   0%(7场)     ← 第8行
```

**每一行由三组数据组成，分属两个不同的维度**：

| 维度 | 数据 | 含义 | 性质 |
|------|------|------|------|
| **本场赔率变化** | `2次 ↑3.6%` | **这场比赛中**0球赔率从初盘到现在总共变了2次，累计上升3.6% | 当前比赛的实时数据 |
| **历史命中率** | `0%(4场)` | **历史上所有比赛中**，0球赔率变化落在"3-4%涨"这个区间的，共4场，其中0场真正打出了0球 | 历史统计验证 |

**⚠️ 这两组数据是完全独立的！**
- `2次 ↑3.6%` 是**这场比赛**的赔率变化情况
- `0%(4场)` 是**历史上**相同变化模式下的实际进球结果
- 两者之间的关系是：**用历史命中率来验证当前赔率变化的真实含义**

**逐字段拆解（以第1行 `0球 2次 ↑3.6% 0%(4场)` 为例）**：

| 字段 | 值 | 含义 |
|------|----|------|
| `0球` | 进球数 | 这一行的统计对象是"总进球=0" |
| `2次` | 变化次数 | 这场比赛中0球赔率总共被庄家调整了2次 |
| `↑` | 变化方向 | 0球赔率在上升（庄家不看好0球，想推离） |
| `→` | 无变化 | 赔率累计变化幅度接近0% |
| `↓` | 变化方向 | 赔率在下降（庄家在防范/诱导该进球数） |
| `3.6%` | 变化幅度 | 0球赔率从初盘到当前累计上升了3.6% |
| `0%` | 历史命中率 | 历史上"0球赔率变化在3-4%涨区间"的比赛中，0场真正打出了0球（0/4=0%） |
| `4场` | 历史样本量 | 历史上符合"0球赔率变化在3-4%涨区间"这个条件的比赛共4场 |

**逐字段拆解（以第2行 `1球 2次 ↑3.8% 5.7%(35场)` 为例）**：

| 字段 | 值 | 含义 |
|------|----|------|
| `1球` | 进球数 | 统计对象是"总进球=1" |
| `2次` | 变化次数 | 1球赔率总共变了2次 |
| `↑3.8%` | 上升3.8% | 1球赔率在上升，庄家推离1球 |
| `5.7%` | 历史命中率 | 历史上"1球赔率在3-4%涨"的35场比赛中，有2场真正打出了1球（2/35=5.7%） |
| `35场` | 历史样本量 | 35场样本量很大，5.7%的命中率非常可靠 |

**逐字段拆解（以第4行 `3球 0次 →0% 30.7%(88场)` 为例）**：

| 字段 | 值 | 含义 |
|------|----|------|
| `3球` | 进球数 | 统计对象是"总进球=3" |
| `0次` | 变化次数 | 3球赔率没有变化！庄家完全没有动3球 |
| `→0%` | 无变化 | 赔率累计变化接近0% |
| `30.7%` | 历史命中率 | 历史上"3球赔率0次变化"的88场比赛中，27场打出了3球（27/88=30.7%） |
| `88场` | 历史样本量 | 88场！超大样本，30.7%极度可靠 |

#### 作用：这个数据怎么用？

**一句话总结：赔率变化告诉你"庄家想让你怎么想"，历史命中率告诉你"历史上实际发生了什么"。用后者验证前者。**

##### 使用场景1：验证推离是否成立（赔率上升时）

```
问题：庄家推离某个进球数（赔率上升），这个推离是真的还是假的？

判断方法：
  → 命中率 ≥ 30% (≥5场) → 推离是假的！该进球数历史上经常打出 → 不能排除
  → 命中率 < 10% (≥5场) → 推离是真的 → 可以排除
  → 命中率 10-30%     → 推离存疑 → 保留观察

举例：
  1球 ↑3.8% + 命中率5.7%(35场)
  → 1球被推离(上升)，但历史命中率只有5.7%，35场中只有2场打出1球
  → 推离有效 → 可以排除1球 ✅
```

##### 使用场景2：验证防范是否成立（赔率下降时）

```
问题：庄家"防范"某个进球数（赔率下降），这个防范是真的还是假的（诱导）？

判断方法：
  → 命中率 = 0% (≥5场) → 假防范！历史上从未打出 → 这不是防范，是诱导！排除
  → 命中率 > 40% (≥5场) → 真防范！庄家确实怕这个方向 → 保留
  → 命中率 0-40%      → 防范存疑 → 保留观察

举例：
```

#### 使用规则

```python
Step 1: 读取每个进球数的变化命中率
  → 命中率 = 该进球数在当前"变化幅度+方向"桶内的历史命中概率

  → 防范保留 + 命中率=0%(≥5场) → ⚠️ "假防范"！修正为排除（强信号）
```

#### 关键逻辑

```
赔率变化命中率的核心价值：
1. "假防范"识别：
   → 赔率看似在降（防范），但历史命中率=0% → 庄家反复用这个手法诱导
   → 例：4球↓8.7% + 命中率0%(11场) → 庄家历史上11次用4球降赔诱导，0次真正打出4球

2. "假推离"识别：
   → 赔率看似在升（推离），但历史命中率很高 → 推离不成立
   → 例：3球↑5.7% + 命中率33.3%(15场) → 推离不成立，保留3球

3. "真防范"确认：
   → 赔率降 + 命中率>40% → 庄家确实在防范这个方向
   → 例：3球↑5.7% + 命中率33.3%(15场) → 中等偏高，保留

4. "真推离"确认：
   → 赔率升 + 命中率<10% → 确认庄家在推离
   → 例：1球↑3.8% + 命中率5.7%(35场) → 命中率极低，确认推离有效

5. ⛔ 样本不足的陷阱（最容易犯的错误！）：
   → 0球 ↑25% + 命中率50%(2场) → 50%看起来很高！但只有2场，1场打出而已
   → 2场中的1次命中 = 可能是巧合，不具备统计意义
   → 正确做法：维持原有的推离判断，不因为"50%"就改主意
   → 反例：如果0球50%(20场) → 20场中10场打出 → 这才叫"命中率高"，可以修正

对比理解：
  50%(2场)  = 2场中1场 → 可能碰巧
  50%(20场) = 20场中10场 → 统计显著
  0%(2场)   = 2场中0场 → 可能只是还没碰到
  0%(20场)  = 20场中0场 → 真的打不出来
```


**推理复盘要点**：

#### 修正优先级

```
赔率变化命中率(4.6) vs 规律命中率统计(4.4) vs 推离判断(4.0-4.3) vs 双命中率交叉(4.6新增)：

1. 双命中率交叉验证 ≥ 单一命中率（双维度互相强化）
2. 赔率变化命中率 ≈ 赔率命中率（独立价值接近，权重赔率×0.6 + 变化×0.4最优）
3. 两者 ≥ 规律命中率统计（同级）
4. 以上 > 赔率变化的单次直觉判断
5. 样本<5场时降权，不覆盖样本≥10场的判断
6. 两个命中率冲突时，取样本更大的那个
```
