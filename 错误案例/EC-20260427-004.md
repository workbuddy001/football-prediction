```

```

## 错误原因分析

### 根本原因：**框架过于依赖"赔率高低"和"隐含概率"，忽略了"变化方向"的一致性**

```
框架推理：
  方法1（赔率高低）：2球3.1最低 → 推荐2球
  方法2（隐含概率）：2球32.3%最高 → 推荐2球
  方法3（变化方向）：2球↓4.6%，3球↓4.2% → 都防范 → 不一致

→ 推荐2球（2个方法一致）
→ 但实际3球 ❌ 错误！
```

---

## 教训

1. **"变化方向+幅度"应该比"赔率高低"权重更高**：
   - 庄家防范的结果（下降）比"赔率最低"更重要
   - 应该增加"变化方向"的权重

2. **框架的"二选一推理方法"权重不合理**：
   - 方法1（赔率高低）权重过高
   - 方法3（变化方向+幅度）权重过低
   - 需要调整权重

3. **系统"排除3球"的理由不充分**：
   - "历史3球率9.1%"不是充分理由
   - 应该结合"变化方向"判断
