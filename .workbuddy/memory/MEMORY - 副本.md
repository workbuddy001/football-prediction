# 足球预测项目长期记忆（纯排除法版）

## 数据文件位置
- **实际比分结果**：`分析模板/_scores.json`（key格式：`4.17_周五001`）
- **比分记录字段**：`home_score` + `away_score`（不是home_goals）
- **比分匹配规则**：必须用match_id匹配。**sporttery_data/*.json中可能有旧数据没有match_id或match_info为空，跳过这些文件**
- **竞彩赔率数据**：`sporttery_data/*.json`（每match_id一个文件）
- **分析模板**：`分析模板/YYYY.MM.DD/编号_主队vs客队_源数据.md`

## 近况数据（Step 0）
- **来源**：sporttery_data中 `preview.recent` 字段（竞彩网自带，主客队各5场历史）
- **覆盖率**：79场比赛中70场(89%)有数据
- **精确得分字段**：`homeTeamFullCourtGoalCnt` / `awayTeamFullCourtGoalCnt`（不要解析fullCourtGoal字符串）
- **判断主客队**：`homeTeamShortName == teamShortName` → 主队；否则客队
- **近况规则**：合并均值<2.0→-8 | 2.0~2.5→-3 | 2.5~3.5→**+3加分**（符合3球特征） | 3.5~4.0→-3 | >4.0→-8

## 术语定义
- **竞彩** = 500.com上标注的"竞彩官方"赔率/赔率变化
- **澳门** = 500.com上标注的"澳门"赔率/澳门心水推荐
- 源数据结构：第二节=初盘（index 0=竞彩, index 2=澳门），第三节=即时盘（同上）
- **⚠️数据提取注意**：initial_odds/realtime_odds数组第一行是注释(# 格式...)，实际index 0是第二行
- **⚠️前端JSON解析陷阱（2026-04-12验证）**：Python后端 `json.dumps(response)` → 前端 `response.json()` 已自动解析为JS对象 → **不要对 res.raw_data 再做 `JSON.parse()`！** 正确做法：`typeof x === 'string' ? JSON.parse(x) : x`
- **⚠️30家赔率遍历索引**：realtime_odds数组从 ri=0 开始遍历（不是ri=1），index 0 就是有效赔率数据

## 大3球预判 + 大球赔率规律（2026-04-23 新增）
- **条件**：预判大3球 + 4球或5球赔率上涨
- **样本**：70场
- **大3球率**：**40.0%**（比大球降时36.3%更高）
- **结论**：当判断大3球时，大球赔率上涨反而是真实信号
- **原理**：赔率涨=被造热，但大球打出的概率仍然更高

### 0球<10 + 高球下降规律（2026-04-23 新增）
**条件**：0球赔率<10 + 高球(5/6/7)赔率下降>-5%

| 条件 | 样本 | 大3球率 | vs基准 |
|------|------|---------|--------|
| 基准(0球<10全部) | 56场 | 12.5% | - |

**结论**：高球下降时大3球概率是对照组的**2.9倍**！庄家压低0球赔率但高球赔率也在降，说明大球概率反而更高。

### 0球<15 + 高球下降 + 2球赔率规律（2026-04-23 新增）
**发现**：用2球赔率作为"攻击力"的代理指标（2球赔率高=攻击力弱），叠加高球下降效果更好

| 条件 | 样本 | 大3球率 | vs基准 |
|------|------|---------|--------|
| 基准(0球<15) | 159 | 23.9% | - |
| **高球降+2球3.5-4.0** | 23 | **34.8%** | **+10.9%** |
| 高球降+2球3.0-3.5 | 49 | 20.4% | -3.5% |
| 无高球下降 | 87 | 23.0% | -0.9% |

**结论**：**2球赔率3.5-4.0区间**是高球下降的最佳叠加条件，大3球率34.8%

**注意**：0球赔率低本身就意味着近况差（86.6%近况<2.0），所以无法用近况分组验证"近况正常+高球下降"的假设

## 大3球 vs 小3球 预判（2026-04-23 新增，2026-04-24 优化）
**文件**：`predict_3goals.py` 的 `predict_big3_vs_small3()` 函数

**背景**：当已有"关注3球"或"排除3球"信号后，进一步判断是：
- 小3球（恰好3球）：3:0, 2:1, 1:2, 0:3
- 大3球（4+球）：4:0, 3:1, 2:2, 1:3, 0:4, 5:0, ...

**核心规律（基于315场回测，33%准确率）**：
| 信号类型 | 条件 | 大3球率 | 小3球率 | 命中率 |
|---------|------|---------|---------|-------|
| 超级3球 | 全局 | 12% | 75% | **75%** |
| 关注3球+0=17 | 特殊 | 0% | 75% | **75%** |
| 关注3球+0=19 | 全局 | 100% | 0% | **83%** |
| 关注3球+0=14（无需黄金3球） | 全局 | **73.3%** | 13.3% | **73.3%** |
| 黄金3球+0=14（需黄金3球） | 全局 | **88.9%** | 0% | **88.9%** |
| 关注3球+0>=22 | 全局 | 50-75% | 0-20% | **60-75%** |
| 关注3球+0=15-16 | 全局 | 40-44% | 20-22% | **40-44%** |
| 关注3球+0=20-21 | <22 | 10-20% | 50-60% | **50-60%** |
| 排除3球 | 全局 | 35-40% | 15-17% | **36%** |
| 排除2球 | +0>=18 | 40% | 15% | **31%** |

**关键发现**：
- 0球=14有**两个规律**：
  1. **关注3球+0球=14**（无需黄金3球）→ 15场，73.3%大3球
  2. **黄金3球+0球=14**（需黄金3球前置）→ 9场，88.9%大3球
- 0球=17是特殊区间，75%小3球
- 0球=19是另一个极端，100%大3球（样本小）
- 0球>=22后倾向大3球
- 0球20-21是中间地带，倾向小3球

**判断阈值**：diff>=10%时给出明确判断（从15%降低）

**前端显示**：sporttery_web.py 已集成，调用 `predict_big3_vs_small3(features, g3_pred=g3_pred, g2_pred=g2_pred)` 返回结果

## 历史赔率命中率统计（2026-04-21 新增，2026-04-22 修正）
- **统计数据来源**：`分析模板/_scores.json` 回测

- **前端显示**：sporttery_web.py 每个赔率旁边显示历史命中率标签（绿色>=35%，黄色20-35%，红色<20%）

### 排除法双选规则（2026-04-22 新增）
**文件**：`predict_3goals.py` 的 `recommend_exclude_double_pick()` 函数
**逻辑**（按优先级）：
1. 0球永远不排除
2. **赔率升>=5% → 强排除**（不管命中率多高，因为赔率上升=被造热）
3. 赔率>3.5 且 命中率<20% → 排除（低概率高赔率）
4. 赔率<=3.5 → 保留
5. 赔率>3.5 且 命中率>=20% 且 赔率没升 → 保留（高赔率但高命中率且未被造热）

**示例**：朗斯vs图卢兹
- 1球 赔率5.0 | 升赔16.3% → 强排除 ✓
- 2球 赔率3.45 | 升赔7.8% → 强排除 ✓
- 3球 赔率3.55 | 升赔0% → 赔率高但命中率>=20%，保留 ✓

### 三条件排除3球（2026-04-22 增强版）
**文件**：`predict_3goals.py` Step 7
**条件（必须全部满足）**：
1. 近况均值 2.5~3.5（正常）
2. 0球赔率 > 13
3. **当前3球 > 3.50**
4. **初始3球也 > 3.50**（关键！初始在黄金区间<3.50的是升赔诱导，不能排除）

**原理**：初始就在3.50以上，说明庄家从一开始就不想要3球；如果初始在黄金区间被升赔推上来的，是诱导信号，不能排除

**计算初始3球**：`初始3球 = 当前3球 / (1 + 变化% / 100)`

**前端显示**：3球赔率上方红色脉冲banner `🚫 排除3球 - 三条件全满足`

# 核心框架：造热原理 + 排除法（纯排除法版）

## 一、底层基础：赔率变化 = 庄家意图

> **庄家通过调整赔率来引导和分流筹码。赔率的每一个变化都反映了庄家想让筹码流向哪个方向、不想到哪个方向。**

| 核心原理 | 含义 |
|---------|------|
| **造热** | 庄家拉低某方向赔率 → 吸引筹码去该方向。单出口=筹码集中，双出口=筹码分流 |
| **推离** | 庄家拉高某方向赔率 → 不想让筹码去那里 = 该方向庄家不愿出 |
| **竞彩×澳门互动** | 同向=信号最强；澳门不动以竞彩为准；竞彩不动以澳门为准；分歧=不确定 |
| **赔率绝对值** | 赔率越高=赔付压力越大=越不愿意该方向打出 |

### 从原理到操作

```
赔率变化 → 读懂庄家意图（造热/推离/不怕/不跟/共识/分歧）
         ↓
    庄家想让什么方向出？不想让什么方向出？
         ↓
    排除"庄家不想让出的方向" = 排除法
```

**排除法是造热原理推导出的操作方法，不是独立于造热的另一个框架。**

---

## 二、排除法：操作框架

> **不预测"什么会出"，而是排除"什么不会出"。三方向中排除越多，剩余方向越可靠。**


### 排除依据（按可靠度排序）

#### ① 心水排除法（★★★★★，最高优先级）
> 先用心水推荐判断排除哪个方向（详见第五节）

#### ② 赔率绝对值排除（★★★★）
| 条件 | 排除 | ⚠️例外 |
|------|------|--------|
| 赔率 > 5.0 | **排除该方向** | 竞彩大幅降该方向>5%时，降赔信号优先（曼城vs皇马 3.18欧冠验证） |
| 赔率 > 3.5 | **大概率排除** | 友谊赛中平赔3.5-5.0不能排除平局（荷兰/西班牙 3.31验证） |
| 赔率 < 1.5 | **排除另外两方向的可能性高** | — |

#### ③ [不怕]标签排除（★★★★）
> 竞彩拉高X方向 + 澳门不动 = 澳门"不怕"X = 庄家笃定X不出

| 条件 | 排除 |
|------|------|
| 竞彩X方向赔率升 + 澳门不动 + **X赔率≥3.5** | **排除X方向** |
| 竞彩X方向赔率升 + 澳门不动 + X赔率<3.5 | **⚠️ 不可靠，不排除**（刚果vs牙买加 平赔3.00失败案例） |

#### ④ [不跟]标签（★★★★）

| 条件 | 判断 |
|------|------|
| 竞彩X方向赔率降 + 澳门不动 | **X方向造热是假象**（不代表X真的会出） |

#### ⑤ 造热排除（★★★★）

| 条件 | 排除 | ⚠️修正 |
|------|------|--------|
| 竞彩X降>5% + 澳门推X + **单出口** | ~~排除X方向~~ | ❌ **已失效**（⑧号：竞彩×澳门同向造热主+推离平客=单出口→平局） |
| 竞彩X降>5% + 澳门推X + **双出口** | ~~不排除~~ | ⚠️ ②③号验证：同向造热+双出口=**实盘信号**，反而是可靠的 |
| **修正后** | | |
| 竞彩×澳门同向造热X + **双出口**（其他方向未全被推离） | **实盘信号，X方向可靠** | ②③号验证 |
| 竞彩×澳门同向造热X + **单出口**（其他两方向全被推离） | **⚠️ 造热陷阱，X方向不可靠** | ⑧号验证 |

#### ⑥ 推离排除（★★★★）
| 条件 | 排除 |
|------|------|
| 竞彩某方向升>5% + 澳门不推 | **排除该方向** |

```
Step 1: 澳门心水排除法（最高优先级）
  ├── 心水方向赔率≥3.5 → ✅ 排除心水方向（80%命中）
  ├── 心水方向赔率3.0-3.5 → ✅ 可排除（75%，样本小）
  ├── 竞彩推离心水方向>3%（升赔）→ ✅ 排除心水方向（规则B，4/4=100%）
  ├── 竞彩降心水方向>2%+单出口 → ❌ 不能排除（实盘信号）
  ├── 心水方向赔率2.0-3.0+竞彩无信号 → ❌ 不能排除（≈随机）
  └── 心水方向赔率<2.0 → 不适用，跳过

Step 2: 赔率绝对值检查
  ├── 某方向赔率>5.0 → 排除
  └── 某方向赔率>3.5 → 大概率排除（友谊赛平局除外）

Step 3: 竞彩×澳门互动检查
  ├── 竞彩X升+澳门不动 → [不怕X] → 排除X
  ├── 竞彩X降+澳门不动 → [不跟X] → X造热假象
  ├── 竞彩X升>5%+澳门不推 → 推离 → 排除X
  ├── 竞彩X降>5%+澳门推X+单出口 → 造热X → 排除X
  └── 竞彩全不动+澳门全不动 → 无信号

Step 4: 基本面检查
  ├── 弱队客场远征 → 排除弱队
  └── 顶级强队碾压（赔率<1.3+FIFA Top 5+对手差距>50位）→ 排除弱队方向

Step 5: 赛事类型调整
  ├── 友谊赛 → 平局概率+，排除法信号降级
  └── 淘汰赛/附加赛 → 90分钟内平局概率+

Step 6: 综合排除结果
  ├── 汇总Step 1-5所有排除结论
  ├── 排除方向之间是否矛盾？ → 矛盾则降级
  └── 剩余方向数量确认

Step 7: 得出结论
  ├── 排除2个 → 只剩1个 → ★★★★★
  ├── 排除1个 → 赔率差>0.5选低赔 → ★★★★
  ├── 排除1个 → 赔率差<0.5优先考虑平局 → ★★★
  ├── 排除1个 → 但友谊赛 → **观望**（除非赔率差>1.0可选低赔）
  ├── 排除1个 → 附加赛防平 → ★★★-★★★★
  ├── 排除不了 → 观望
  └── 多标签冲突 → 信号矛盾 → 观望
```

## 四、排除1个后的选择标准

| 条件 | 选择 | 原理 |
|------|------|------|
| 剩余赔率差>0.5 | 选低赔方 | 庄家更能承受低赔方打出 |
| 剩余赔率差<0.5 | **优先考虑平局** | 两方向赔付接近→平局安全 |
| 友谊赛 + 排除1个 | **⚠️ 观望**（除非赔率差>1.0可选低赔） | 友谊赛中选低赔/防平都不可靠（墨西哥1-1、巴西3-1验证） |
| 附加赛 + 平赔<3.5 | **考虑平局** | 附加赛90分钟平局概率高 |

## 五、心水排除法（Step 1，最高优先级）

> ⚠️ **总体命中率36.4%（33场）**，低于随机，**但特定条件下排除心水方向命中率极高**

### 心水赔率区间与排除规则

| 心水赔率区间 | 排除正确率 | 规则 |
|-------------|-----------|------|
| **≥3.5** | **80%**（10场8不出） | ⭐ **可独立排除心水方向** |
| **3.0-3.5** | **75%**（4场3不出） | ⭐⭐ 可排除，样本小 |
| **2.0-3.0** | **54.5%** | ❌ **不能单独排除**，须配合竞彩信号 |
| **<2.0** | **50%** | ❌ 不能排除（友谊赛中更低） |

### 心水×竞彩信号体系（非友谊赛 + 心水>2.0前提下）

| 竞彩对心水方向 | 操作 | 命中率 | 原理 |
|---------------|------|--------|------|
| **升>3%**（推离心水） | **✅ 排除心水**（规则B） | **4/4=100%** | 心水说买X+赔率拉高X=言行不一=X不出 |
| **降>2% + 单出口** | **❌ 不能排除**（实盘） | **0/2=0%** | 竞彩单独造热X=看好X=心水是真信号 |
| **降>2% + 双出口** | **⚠️ 可排除** | **1/1=100%**（极小样本） | 筹码分流→X难出 |
| 变化<2%或不动 | **⚠️ 排除心水**（规则A） | **7/14=50%** ⚠️降级 | 竞彩不动≠信号 |

### 心水>2.0排除→选低赔
| 条件 | 场次 | 命中率 |
|------|------|--------|
| 全部（19场） | 19 | 52.6% |
| **排除友谊赛** | **10** | **70%** |
| 友谊赛 | 9 | 33.3% ❌ |

**使用条件**：①非友谊赛 ②排除后赔率差>0.5 ③近况差不强烈支持心水方向（≥8时降级为观望）
**近况差辅助反而降低命中率**（36.8%），排除后直接选低赔即可

### 心水排除三层验证
1. **庄家信号层**：心水+赔率变化
2. **筹码格局层**：单出口/双出口
3. **基本面层**：球队近期状态
> **三层一致 = 最可靠** | 信号层排除但基本面不支持 = 降级

## 九、关键教训

1. ⚠️ **友谊赛是最大障碍**（3.31最终验证）：15场友谊赛V7命中率66.7%，排除1个后选低赔/防平都不可靠。**友谊赛只能排除1个时应观望**
2. ⚠️ **友谊赛排除1个后赔率差>1.0才可选低赔**（巴西3-1：主1.59/平3.75差2.16→应选主胜而非防平）
3. ⚠️ **平局是最大盲区**：排除1个方向后不能默认选另一方向，必须评估平局概率
4. ⚠️ **友谊赛中平赔3.5-5.0排除不可靠**（荷兰/西班牙验证）
5. ⚠️ **[不怕X]赔率<3.5不可靠**（刚果vs牙买加 平赔3.00验证）
6. ⚠️ **观望后仍可进一步分析**：友谊赛排除1个后剩2个，仍应优先考虑平局
7. ⚠️ **竞彩单方大幅造热>5%时**：澳门小幅反向时竞彩方向可能仍有效
8. ⚠️ **规律M（顶级强队实盘）必须3条同时满足**：①赔率<1.3 ②FIFA Top 5 ③对手差距>50位
9. ⚠️ **弱队基本面碾压不是造热陷阱**：库拉索排名60+客场远征=正常碾压
10. ⚠️ **无信号时不投注**：无信号+基本面判断3场全错
11. ⚠️ **近况差不用于选方向**：排除心水后直接选低赔，近况差仅用于检测矛盾
12. ⚠️ **规则B（竞彩推离心水>3%）累计5/5=100%**（4.02①号验证）：但排除后选择需谨慎
13. ⚠️ **排除后低赔方向基本面极端差→优先考虑平局**（4.02②③验证）：排除客后主队8轮不胜→实际平局。这是对"赔率差>0.5选低赔"的重要补充——当低赔方向基本面极端差时，平局更安全
14. ⚠️ **心水推"和局"且赔率≥3.5时，不能排除平局**（4.03①验证）：心水推平赔率3.73→排除平→实际平局。心水推平本身就说明庄家认为平局概率高，不能简单用赔率绝对值排除
15. ⚠️ **排除某方向的前提是该方向有赔率信号（推离/造热）**（4.03①深化）：001号主（造热）和客（推离）都有明确信号，但平局赔率几乎不动（竞彩+0.8%/澳门0%）=庄家对平局无表态=不能排除。**没有信号的方向不能排除**
16. ⚠️ **主客剧烈震荡+平局不动+心水推平=庄家转移平局筹码**（4.03①深化）：庄家在主/客制造大幅变化吸引大众关注主或客，而平局赔率不动+心水推平=庄家真正看好平局但不希望筹码流向平局
17. ⚠️ **竞彩×澳门三方向全面分歧+基本面一面倒=庄家掩护平局**（4.03⑤验证）：诺维奇近况83.3碾压客队5.6+客6连败，但竞彩×澳门全部分歧→1-1
18. ⚠️ **排除客后选低赔主胜失败模式**（4.03⑤）：存在掩护信号时应优先考虑平局而非选低赔
19. ⚠️ **巨幅造热判别需结合出口结构**（4.03②③vs⑧对比）：双出口同向造热=实盘；单出口全面造热=陷阱
20. ⚠️ **心水推低赔方向可靠**（4.03④验证）：无信号时心水本身有价值
21. ⚠️ **掩护平局模式确认有效**（4.03⑤⑥验证）：基本面一面倒+平赔不动=庄家掩护平局
22. ⚠️ **观望策略价值极高**（4.03验证）：6场中4场观望全部安全
23. ⚠️ **推离平≠排除平**（4.03⑧验证）：圣旺红星推离平→实际0-0平局
24. ⚠️ **出口结构是造热/陷阱判别的核心**（4.03⑧总结）
25. ⚠️ **心水推X+澳门造热X(降赔)=烟雾弹，排除X**（4.03⑧⑪⑦三次验证）。⚠️必须是澳门造热X才适用
26. ⚠️ **[不跟]≠排除**（4.03⑨验证）：只是说信号假象，不代表该方向不会出
27. ⚠️ **竞彩几乎不动=实盘**（4.03⑪验证）：竞彩变化<2%=不操盘=按实力打=基本面说了算
28. ⚠️ **不确定时不要强行排除**（4.03⑨验证）：没有依据时观望比强行选方向更安全
29. ⚠️ **掩护模式**（4.03①⑥⑬验证）：筹码被吸引去某方向→另一方向安全打出
30. ⚠️ **规律B=实盘+联赛实力差距小→"碾压"不可靠**（4.04②验证）：澳超等实力接近的联赛"碾压"更多是短期波动
31. ⚠️ **规律B=实盘时要综合所有基本面因素**（4.04①验证）：不仅看近况差还要看对赛/主场优势
32. ⚠️ **赔率数据源可能不是最终收盘赔率**（4.04①验证）：应尽量使用最终收盘赔率
33. ⚠️ **让球后赔率框架已废弃**（4.05用户决定）：用户判断"让球造热分散筹码"框架不成熟，要求回退到纯排除法
34. ⚠️ **单出口+中水不能直接判定为诱盘**（4.13验证）：必须用标准盘做让球×不让球交叉验证
35. ⚠️ **三重共振（排除法×让球出口×基本面）=最大陷阱**（墨尔本城vs惠灵顿验证）：越完美共振=庄家引导效果越好
36. ⚠️ **庄家最优解=赔付压力最小的方向**：看标准盘哪个赔率最低，那才是庄家真正保护的方向
37. ⚠️ **不让球赔率与让球赔率是两套独立数据源**：`jc_odds`=让球盘赔率，`standard_odds`(realtime_odds[0])=标准1X2赔率

## 十、竞彩让球赔率水位体系（2026-04-13确立）

> **此分类专门针对竞彩让球赔率（如主让-1球、客让+1球的胜平负赔率），与普通欧赔是两套独立体系。**

```
超高水(>3.5) ── 高水(2.8~<3.5) ── 中庸(2.45~<2.8) ── 中水(1.80~<2.45) ── 低水(1.50~<1.80) ── 超低水(<1.50)
    │                  │                 │                │                 │               │
   阻！               诱！              散             偏向            默认            确信
望而却步          博高倍心态        无所适从        容易偏向        容易打出       概率很大
```

| 档位 | 赔率范围 | 庄家意图 | 玩家心理 | 关键词 |
|------|---------|---------|---------|--------|
| **超高水** | **>3.5** | **阻** — 拉高让你不敢买 | 太高了不值得博 | **望而却步** |
| **高水** | **2.8 ~ <3.5** | **诱** — 用高倍勾你博 | 博一搏单车变摩托 | **高倍诱惑** |
| **中庸** | **2.45 ~ <2.8** | **散** — 让你拿不准 | 好像有可能又没有 | **无所适从、分散** |
| **中水** | **1.80 ~ <2.45** | **引** — 合理区间引导 | 这个看起来靠谱 | **比较容易偏向** |
| **低水** | **1.50 ~ <1.80** | **确认** — 低赔=大概率 | 这个应该能出 | **默认容易打出** |
| **超低水** | **<1.50** | **强确认** — 几乎确定 | 稳了 | **概率很大** |

> **核心区分**：3.55（超高水/阻）vs 3.00（高水/诱）——看起来都是"高赔"，但庄家意图截然相反！

## 十一、让球×不让球交叉验证框架（2026-04-13确立）

> **核心思想：排除法结论不能单独信任，必须用标准盘（不让球）做交叉验证，检验庄家最优解是否在反向。**

### 触发条件
当**让球盘呈现单出口结构**且预测方向落在**中水区(1.80~2.45)**时启动交叉验证。

### 交叉验证三步法

```
Step 1: 让球盘出口分析
   单出口 → 筹码被迫集中到唯一可买方向
         → 如果该方向=排除法结论方向 ⚠️

Step 2: 标准盘赔付压力检验
   预测方向的标准盘赔率：
   - <1.80（低水/超低水）→ 庄家真正保护的方向 ✅
   - 1.80~2.45（中水）→ 中性
   - >2.8（高/超高水）→ 赔付压力大 ❌
   
   非预测方向的赔率是否有≥2.8的高赔付？
   有 → 庄家不想这些方向出 → 排除法结论方向可能被诱导

Step 3: 基本面共振检验
   基本面倾向是否与排除法结论一致？
   一致 → 三重共振=筹码最集中=最大陷阱嫌疑 ⚠️⚠️⚠️
   分歧 → 降低诱导概率 ✅
```

### 判定等级表

| 判定 | 条件 | 含义 |
|------|------|------|
| 🔥 **三重共振陷阱** | 单出口+中水 + 标准盘异常 + 基本面共振 | 庄家最优解在**反向**！给出反向建议 |
| 🧲 **交叉验证异常** | 单出口+中水 + 标准盘异常(非预测方高赔/预测方低水) | 高度怀疑诱导 |
| ⚠️ **单出口诱导怀疑** | 单出口+中水 + 标准盘无异常 | 降级处理 |
| ✅ **基本面分歧降级** | 单出口+中水 + 基本面不支持预测方向 | 较安全 |
| ✅ **双出口/实盘防守** | 多个方向水位合理 / 低超低水单出口 | 可信 |

### 教学案例：墨尔本城 vs 惠灵顿（2026-04-12）
```
排除法结论：平局 ★★★★★
澳门心水：和局 → 平局
基本面：客队+5分优势 → 客优

让球盘(-1)：主胜3.00(高水诱) / 平3.55(超高水阻) / 客胜1.95(中水)
  → 单出口！筹码集中到客不败

标准盘：主胜1.46(低水)↑13% / 平3.75(高水)↓7.4% / 客3.90(高水)↓22.8%
  → 平和客都在高水区间(>2.8)，赔付压力巨大
  → 主胜1.46 = 全场最低 = 庄家最小赔付选项！

三重共振：排除法(平) × 让球出口(客不败) × 基本+5(客优) = 完美共振
  → 但标准盘揭示主胜才是庄家最优解

最终结果：主队2-0打出 ✅
```

### 核心教训
- **⭐ 单出口+中水≠一定诱盘**：必须做交叉验证
- **⭐ 三重共振=最危险信号**：越完美的共振=越好的引导效果
- **⭐ 庄家最优解=赔付压力最小的方向**（标准盘最低赔率）
- **⭐ 不让球高赔率方向=庄家不愿出的方向**（但要看是阻还是诱）

### 常见陷阱

1. **recent 是 dict 不是 list**：用 `recent.get('home', {}).get('matchList', [])`
2. **变量名复用**：循环内的 `key` 不要与外层的 `scores_by_key` 键名混淆
3. **ttg_c 是 dict**：要用 `change.get('change_pct', 0)` 获取变化率
4. **match_id 匹配**：`_scores.json` 的 key 有时是 "2038722"，有时是 "4.08_周三001"
