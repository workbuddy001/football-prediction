# 子集规则挖掘系统 — 需求与设计文档

> 版本: v1.0 | 日期: 2026-06-04 | 目标读者: 执行AI

---

## 一、系统目标

在现有**全量规则挖掘系统**（`_rule_mining_step1~3`）的基础上，增加**子集维度**的规则挖掘能力。按联赛/比赛类型/其他标签筛选数据子集后，独立进行特征提取和网格搜索，发现子集特有的投注规律。

**核心价值**：
- 五大联赛 vs 小联赛的进球模式差异巨大
- 国家队比赛（国际赛、世预赛、欧冠）与俱乐部联赛的结构不同
- 某些规律只存在于特定联赛（如日职联的小球倾向）

## 五、各模块详细设计

### 5.1 Step 0: 数据标注模块 (`_subset_tag_data.py`)

**功能**：给 `_feature_matrix.csv` 追加联赛和分类标签列。

**输入**：
- `_feature_matrix.csv`（870行 x 94列）
- `sporttery_data/{match_id}.json`（含 preview）
- `分析模板/_scores.json`（含 league）

**输出**：
- `_feature_matrix_tagged.csv`（870行 x 100列，新增6列分类标签）

**新增列**：

| 列名 | 类型 | 示例值 |
|------|------|--------|
| `league` | string | `"英格兰超级联赛"` |
| `match_type` | string | `"联赛"` / `"杯赛"` / `"国际赛"` |
| `tier` | string | `"顶级联赛"` / `"次级联赛"` / `"杯赛"` / `"国际赛"` |
| `region` | string | `"欧洲"` / `"亚洲"` / `"美洲"` / `"国际"` |
| `league_group` | string | `"五大联赛"` / `"北欧联赛"` / ... |
| `league_subgroup` | string | `"英格兰超级联赛"`（联赛名本身，用于单独联赛挖掘） |

**联赛名获取优先级**：
1. 先查 `_scores.json` 中 `match_id` 对应条目的 `league` 字段
2. 若无，读取 `sporttery_data/{match_id}.json` 的 `match_info.league`
3. 仍无，从 `preview.tables.tournamentShortName` 或 `preview.recent` 中推断
4. 最终无 → `league = "未知"`, `league_group = "未分类"`
5. **关键过滤**：只保留 `league != "未知"` 的行 → 排除无法分类的比赛

**分类规则**：
```python
# match_type 判定
if league in ["国际赛", "世界杯预选赛"]:
    match_type = "国际赛"
elif "杯" in league or "冠军联赛" in league or "联赛杯" in league:
    match_type = "杯赛"
else:
    match_type = "联赛"

# tier 判定
if league in TOP5_LEAGUES:
    tier = "顶级联赛"
elif league in SECOND_TIER:
    tier = "次级联赛"
elif match_type == "杯赛":
    tier = "杯赛"
elif match_type == "国际赛":
    tier = "国际赛"
else:
    tier = "其他联赛"
```

### 5.2 Step 1s: 子集特征提取 (`_subset_extract.py`)

**功能**：按分类标签从 `_feature_matrix_tagged.csv` 中拆分出各子集的独立CSV。

**输入**：
- `_feature_matrix_tagged.csv`

**输出**：
- `subset_data/五大联赛.csv`（330+行）
- `subset_data/英格兰超级联赛.csv`（62行）
- `subset_data/西班牙甲级联赛.csv`（90行）
- `subset_data/国家队赛事.csv`（46行）
- `subset_data/北欧联赛.csv`（115行）
- ... 按需生成

**核心代码逻辑**：
```python
import pandas as pd

df = pd.read_csv('_feature_matrix_tagged.csv')

SUBSET_CONFIGS = {
    "五大联赛": {"league_group": ["五大联赛"]},
    "英超": {"league_subgroup": ["英格兰超级联赛"]},
    "西甲": {"league_subgroup": ["西班牙甲级联赛"]},
    "德甲": {"league_subgroup": ["德国甲级联赛"]},
    "意甲": {"league_subgroup": ["意大利甲级联赛"]},
    "法甲": {"league_subgroup": ["法国甲级联赛"]},
    "五大次级联赛": {"league_group": ["五大次级联赛"]},
    "北欧联赛": {"league_group": ["北欧联赛"]},
    "亚洲联赛": {"league_group": ["亚洲联赛"]},
    "欧战": {"league_group": ["欧战"]},
    "国家队赛事": {"league_group": ["国家队赛事"]},
    "国内杯赛": {"league_group": ["国内杯赛"]},
}

os.makedirs('subset_data', exist_ok=True)
for name, config in SUBSET_CONFIGS.items():
    sub = df.copy()
    for col, values in config.items():
        sub = sub[sub[col].isin(values)]
    if len(sub) >= 10:  # 最少10场才有挖掘意义
        sub.to_csv(f'subset_data/{name}.csv', index=False)
        print(f'{name}: {len(sub)}场')
```

### 5.3 Step 2s: 子集网格搜索 (`_subset_gridsearch.py`)

**功能**：对每个子集CSV，复用现有多维度网格搜索逻辑。

**重要**：不要重写网格搜索代码，而是导入现有 `_rule_mining_step2_gridsearch.py` 中的核心函数，只换数据源。

**输入**：
- `subset_data/*.csv`

**输出**：
- `subset_data/五大联赛_candidates.json`
- `subset_data/英超_candidates.json`
- ...

**候选规则JSON格式**（与现有 `_candidate_rules.json` 一致）：
```json
[
  {
    "target_goal": 3,
    "conditions": "g0_[9-11) & ch_g3_dn20+",
    "n": 12,
    "hit": 5,
    "hit_rate": 41.7,
    "avg_odds": 3.50,
    "roi": 45.8,
    "rule_id": "SUB-001"
  }
]
```

**样本量不足的子集处理**：
- 样本 >= 30 → 正常网格搜索，结果标记 `quality: "good"`
- 样本 15-29 → 网格搜索但降置信，结果标记 `quality: "moderate"`，只输出 Top 5
- 样本 10-14 → 不做网格搜索，仅输出描述性统计（各进球数命中率分布），标记 `quality: "insufficient"`
- 样本 < 10 → 跳过

### 5.4 Step 3s: 对比分析引擎 (`_subset_compare.py`)

这是本系统最核心的模块。**子集规则的价值不在于"发现了什么"，而在于"发现了什么全量规则没有的"**。

**功能**：对比子集候选规则与全量候选规则，标记差异点。

**对比维度**：

#### A. 规则排他性评分
```
排他性 = 子集命中率 / 全量同规则命中率 - 1
```
- 排他性 > 50% → **强子集特化**（只在子集中有效）
- 排他性 20-50% → **弱子集特化**
- 排他性 -20~20% → **通用规律**（与全量一致）
- 排他性 < -20% → **反向规律**（子集中反而无效）

#### B. 全量规则在子集中的表现
逐条评估现有14条规则在每个子集中的命中率。这比挖掘新规则更直接有用。

#### C. 与现有规则冲突检测
子集新规则是否与现有全量规则矛盾？如有，优先信哪个？

**输出**：
```json
{
  "subset": "英格兰超级联赛",
  "n": 62,
  "exclusive_rules": [
    {
      "rule_id": "SUB-PL-001",
      "condition": "g3_[3.0-3.5) & ch_had_w_dn10-20",
      "subset_hitrate": 55.6,
      "full_hitrate": 22.1,
      "exclusivity": 151.6,
      "quality": "strong_exclusive"
    }
  ],
  "full_rules_in_subset": {
    "G2": {"hitrate": 45.0, "roi": 180.0, "n": 5, "vs_full": "+4%"},
    "S7": {"hitrate": 61.5, "roi": 220.0, "n": 8, "vs_full": "+12%"},
    ...
  },
  "new_signals": [...],
  "warnings": [...]
}
```

## 六、样本量处理策略

这是子集挖掘的核心挑战。以下为必须遵守的策略：

| 样本量 | 行动 | 输出 |
|--------|------|------|
| >= 100 | 正常网格搜索（2特征组合） | Top 20候选 |
| 50-99 | 简化网格搜索（仅1特征） | Top 10候选 |
| 30-49 | 简化网格搜索（仅核心特征） | Top 5候选 + 描述性统计 |
| 15-29 | 不做网格搜索 | 仅描述性统计 + 全量规则表现 |
| < 15 | 跳过 | 不做任何分析 |

**核心特征列表**（小样本时使用）：
```python
CORE_FEATURES = [
    'g0', 'g1', 'g2', 'g3',           # 进球赔率
    'ch_g0_pct', 'ch_g1_pct', 'ch_g2_pct', 'ch_g3_pct',  # 赔率变化
    'form_avg',                         # 近况均值
    'hhad_is_home_give', 'hhad_is_guest_give',  # 让球方向
    'had_w', 'had_d', 'had_l',         # 胜平负赔率
]
```

**统计显著性要求**：
- 所有候选规则必须标注 `wilson_ci_lower`（Wilson置信区间下界）
- `wilson_ci_lower < 20%` 且 `n < 10` → 标记 `⚠️ 样本不足，不建议实盘`

## 八、与现有全量规则的对比检查清单

执行AI在产出报告前，必须逐项回答以下问题：

- [ ] 此子集中，**哪些全量规则表现显著优于全量平均水平**？（ROI差异>+50%）
- [ ] 此子集中，**哪些全量规则表现显著差于全量平均水平**？（ROI差异<-30%）
- [ ] 此子集中，**哪个进球数的命中率与全量分布偏差最大**？
- [ ] 是否有**赔率区间在全量中可靠、在此子集中失效**？
- [ ] 是否有**赔率变化方向在全量中没有信号、在此子集中成为强信号**？

## 十一、关键注意事项（给执行AI）

1. **联赛标签缺失的行要排除**：`_feature_matrix.csv` 有870行，但部分 match_id 无法匹配到联赛标签。这些行在子集分析中标记 `league="未知"`，只在全量分析中使用。

2. **样本量是硬约束**：网格搜索至少需要15场数据。对于 <15 场的子集，只做描述性统计。不要因为某个条件"看起来合理"就降低样本门槛。

3. **排他性不是唯一标准**：一个规则排他性高 + n=5，不如排他性中等 + n=30 可靠。**优先推荐样本量充足的规律**。

4. **不要造数据**：如果某个子集数据不足，坦诚标注"样本不足，无法给出可靠结论"。

5. **与现有规则的交叉触发必须检查**：如果子集新规则的条件与现有规则高度重叠，报告必须明确指出潜在冲突。

6. **Wilson置信区间是必须的**：所有命中率必须附带95% Wilson CI。小样本下点估计不可靠。

7. **报告语言用中文**：所有产出物使用中文。

## 附录B：Wilson置信区间公式

```python
import math

def wilson_ci(hits, n, z=1.96):
    """Wilson score interval for binomial proportion"""
    if n == 0:
        return (0, 1)
    p = hits / n
    denom = 1 + z*z/n
    center = (p + z*z/(2*n)) / denom
    margin = z * math.sqrt((p*(1-p) + z*z/(4*n)) / n) / denom
    return (max(0, center - margin), min(1, center + margin))
```
